## Summary

- Kept both sign-in methods on the login screen, but shifted the UX toward the approved teacher-friendly flow by making `DepEd Google` the recommended default, keeping `Email & Password` as the fallback path, removing the consent checkbox gate, and centralizing clearer login/OAuth error copy.
- Continued refining the login experience after the initial rollout by turning manual sign-in into a separate animated card state, auto-switching into that view after Google OAuth failures, moving the return action to a top-left back button, rebuilding the fallback copy into a smaller secondary prompt, and making the login shell feel calmer but more intentional.
- Simplified the main school-user dashboard language so teachers see `AIP - Annual Plan` and `PIR - Quarterly Report`, removed the redundant welcome subtitle, clarified the locked PIR state as `Submit AIP first`, and added tour anchors to the action prompt, module cards, and submission history.
- Reworked the AIP and PIR form experience with plainer validation, draft-restore, success, failure, and final-confirmation messages; added sticky bottom wizard navigation on mobile; added bottom spacing so controls do not cover content; and converted the denser full-mode table sections into card-friendly small-screen layouts.
- Added shared help and onboarding infrastructure with a new in-app `/getting-started` page rendered from the existing teacher guide, a floating `HelpLauncher`, a reusable `OnboardingTour`, and route-aware `portalHelpConfig` entries for the user dashboard, AIP, PIR, CES, Cluster Head, and Admin portals.
- Added role-specific tour targets across teacher, CES, Cluster Head, and Admin screens so onboarding can point to queue filters, review actions, sidebar navigation, notifications, autosave status, step navigation, and review/submit areas without relying on brittle selectors.
- Added scheduled in-app deadline reminders on the backend by reusing the existing notification and SSE pipeline, so school and division users can receive quarter-deadline reminder notifications inside the portal even though outbound email/SMS delivery is still not wired in this batch.
- Reintroduced the facade photo and visual atmosphere on the login page with lighter veiling, soft gradient bands, and subtle color glows, then replaced the stripped-down footer with a more deliberate institutional dock that brings back DepEd branding, portal identity, and a single SDO IT help action without returning to the original noisy social/contact layout.
