## Summary

- Replaced Observer role with Cluster Consultant — a cluster-scoped monitoring role that can view and comment on PIRs within their assigned cluster.
- Added PIRComment model, PIR edit-request fields (edit_requested, edit_requested_at, edit_request_count), and cluster_consultant_revision_flow migration.
- Created dedicated /api/cluster-consultant and /api/program-owner route modules with cluster-scoped PIR list, detail, and comment endpoints.
- Removed all Observer guard references; renamed guards to adminDivisionPersonnelOrCES/adminDivisionPersonnelOrSuperintendent variants.
- Updated frontend auth, sidebar, user form (with cluster selector), role theme, onboarding, tour chapters, and changelog to reflect Cluster Consultant role.
